Data (scans) widget guide

 

This widget is used to load multiple scans located in a folder. It loads all recognized filenames and groups them by extension. The widget has many options to order filenames and split them into different scans, as seen in the below image.

The option 'select file reader' is used to select a files with specific extension. Only same type of files can be loaded using this widget. The option 'number of scans' is used to split the list of filenames into multiple scans. Filenames can be sorted in ascending order with timestamp or in alphabetical order, or using sort selection.

E.g. using sort selection '|_|.has', the files are sorted by the strings in the location of '|'. If the data is ['forward_1.has', 'forward_2.has', 'backward_1.has'], it is sorted by 'forward'/'backward' and after by the number after character '_'. Since backward scans should be loaded in reverse (descending) order, an option to flip sequence (ascending to descending) is also implemented.

The loaded data at the output channel is in dictionary format, in the following format.

output data =
{
     scan_data : {
                           scan_0 : {'height':Array(m,n)},
                           scan_1 : {'height':Array(m,n)},...
                       }
}
